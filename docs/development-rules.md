# Our Development Rules

These are our "laws." They'll keep the project healthy.

## Rule 1

The app should always build.

## Rule 2

The app should always run.

## Rule 3

Every feature gets committed separately.

## Rule 4

Never hard-code event data.

## Rule 5

Document every major decision.

## Rule 6

Keep the interface simple enough that a volunteer can learn it in under five minutes.
